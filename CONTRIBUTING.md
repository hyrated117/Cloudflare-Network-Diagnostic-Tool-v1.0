# CONTRIBUTING

Thank you for considering contributing! This project follows a clean, predictable workflow to keep quality high.

---

## 1. Getting Started

1. Fork the repository  
2. Clone your fork  
3. Create a feature branch:  
   `git checkout -b feature/my-feature`

---

## 2. Development Standards

- Follow existing code style
- Write clear, descriptive commit messages
- Add tests for new logic
- Update documentation when behaviour changes

---

## 3. Pull Requests

- PRs must target `main`
- Include a summary of changes
- Ensure CI passes (build, lint, tests)
- Keep PRs focused and small

---

## 4. Issue Reporting

When opening an issue, include:

- Expected behaviour  
- Actual behaviour  
- Steps to reproduce  
- Logs or screenshots (if relevant)

---

## 5. Code of Conduct

Be respectful, constructive, and collaborative. No harassment or toxicity.
